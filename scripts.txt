2.2 Category 2 — Client–Server Application Using Sockets
We will cover sockets and select in week 10, so the challenge with selecting this category is
that you will be learning about sockets while starting work on your project.
In this category your program implements a TCP client–server application. One process
acts as the server; one or more client processes connect over the network (or over the loopback
interface for local testing).
Design requirements
• The server must handle multiple clients concurrently using a single call to select
(or poll) inside the main event loop. Threads and fork-per-client are not permitted
as the primary concurrency mechanism.
• The server must never block on a single client. Reading from or writing to any one
client must not prevent the server from servicing other clients.
• The server must handle client disconnection gracefully: close the socket, remove it
from the fd set, and free all associated resources.
